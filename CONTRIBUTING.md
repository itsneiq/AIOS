# Contributing to AIOS

Thank you for your interest in contributing to AIOS.

AIOS is a specification-first repository. Contributions should preserve SSOT authority, version compatibility, and document consistency.

## Contribution Rules

- Do not redefine AIOS Core in derived documents.
- Keep Runtime changes within runtime behavior.
- Keep Production changes within production behavior.
- Use issues or RFCs for major changes.
- Keep document versions and references synchronized.
- Update changelog entries when applicable.

## Workflow

1. Open an issue or RFC for significant changes.
2. Create a branch for the change.
3. Update only the relevant document area.
4. Run an editorial review.
5. Submit a pull request.

## Commit Style

Recommended prefixes:

- `feat:` for new specification behavior.
- `docs:` for documentation changes.
- `audit:` for audit or consistency checks.
- `release:` for release preparation.
- `chore:` for repository maintenance.

## Review Criteria

A contribution should be reviewed for:

- authority alignment;
- version consistency;
- cross-reference correctness;
- editorial clarity;
- release compatibility.
