# AIOS Documentation Set

This folder contains the AIOS specification suite and supporting documentation.

## Contents

- Core — canonical architectural definitions and normative terminology
- Runtime — runtime behaviors, initialization, and orchestration requirements
- Production — production preparation and operating-system behaviors
- Prompt Standards — reusable prompt structure and authoring guidance
- Templates — canonical templates and output definitions
- Repository — governance, lifecycle, contribution, and ownership guidance
- QC — validation workflow and quality gate standards
- Export — packaging and export rules
- RFC — proposal templates and change discussions
- CHANGELOG — release history and sprint summaries

## Reading order

1. Start with Core to understand the canonical definitions.
2. Continue with Runtime and Production for implementation behavior.
3. Use QC, Repository, and Export materials for quality, governance, and release concerns.

## Rule of thumb

Treat the Core specification as authoritative. Lower-level documents should reference it rather than redefine it.
